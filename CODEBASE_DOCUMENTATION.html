🔌 API Layer & Services

Authentication Service

Method Endpoint Description
handleLogin() POST /auth/login User login with email & password → returns access_token
registerUser() POST /auth/register Create new user account
getProfile() GET /user/profile Fetch current user profile (requires token)
updateProfile() PUT /user/profile Update user information

Problem Service (Most Comprehensive)

Method Endpoint Description
getAllProblems() GET /problems Fetch all problems with pagination
getProblemById() GET /problems/:id Full problem details (creator, stubs, test cases)
createProblem() POST /problems Create new problem (requires auth)
updateProblem() PUT /problems/:id Update problem metadata
deleteProblem() DELETE /problems/:id Remove problem permanently
getMyCreatedProblems() GET /user/problems Fetch user's created problems
createCodeStub() POST /code-stubs Add code template for language
updateCodeStub() PUT /code-stubs/:id Modify language code template
createTestCase() POST /test-cases Add test case (input/output)
updateTestCase() PUT /test-cases/:id Modify test case
deleteTestCase() DELETE /test-cases/:id Remove test case

Article Service

Method Endpoint Description
getAllArticles() GET /articles Fetch all published articles
getArticleBySlug() GET /articles/:slug Get article details by slug
createArticle() POST /articles Publish new article (requires auth)
getTopAuthors() GET /authors/top Leaderboard of top authors

Post Service (Social)

Method Description

🔐 Authentication Architecture

User enters credentials (email/password)
↓
handleLogin() API call
↓
Server returns JWT token
↓
Store token in Zustand auth-store
↓
AuthGate validates token on mount
↓
Fetch user profile & set authenticated state
↓
Token attached to all protected API calls
↓
Protected routes accessible to authenticated users

Token Management

  • Storage: Zustand store with localStorage persistence
  • Persistence Key: "auth-store"
  • Header Format: Authorization: Bearer {token}
  • Validation: Happens in AuthGate component on app load
  • Hydration: Safe SSR handling with _hasHydrated flag

📦 State Management

Zustand Stores

auth-store.ts

  • isAuthenticated: boolean
  • token: string | null
  • user: IUser | null
  • _hasHydrated: boolean
  • Persisted: localStorage

problem-store.ts

  • mode: "create" | "edit"
  • currentProblemInEditor: IProblemData | null
  • codeStubsState: Record<language, code>
  • testCasesState: { id, input, output }[]
  • Persisted: localStorage

React Query (Server State)

Manages server-side data with automatic caching:

  • useQuery() - Fetch & cache data
  • useMutation() - Create/update/delete
  • QueryClient - Centralized cache
  • Auto-refetching on focus
  • Request deduplication
  • Error & loading states

Example:

const { data, isLoading } = useQuery({ queryKey: ['problems'], queryFn: getAllProblems })

💡 Key Architectural Insights

1. Feature-Based Organization

Components are organized by feature (article, challenge, problems, etc.) making it easy to maintain and extend individual features independently.

2. Separation of Concerns

Clear separation between API layer, state management, components, and utilities. Services handle all backend communication, stores manage global state, and components focus on rendering.

3. Type-Safe Development

Comprehensive TypeScript definitions for all API responses and data models. Strict mode enabled for maximum type safety and IDE support.

4. Authentication Flow

Token-based JWT authentication with automatic persistence, validation on mount, and automatic injection into all API calls. AuthGate provides route protection.

5. Performance Optimizations

React Compiler for auto-memoization, React Query for request deduplication and caching, code splitting via App Router, and image optimization via CDN.

6. State Management Strategy

Hybrid approach: Zustand for persistent client state (auth, problem editor), React Query for server state. Each serves its purpose effectively.

7. Rich Code Editor Experience

Full-featured CodeMirror integration with syntax highlighting for 4 languages, theme switching, and integration with test case evaluation system.

8. Responsive & Accessible Design

Mobile-responsive layouts, dark/light mode support, Radix UI for accessibility, and comprehensive component library for consistent UX.
